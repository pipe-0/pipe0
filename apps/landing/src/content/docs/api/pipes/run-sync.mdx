Enrich data synchronously. This endpoint waits for the enrichment to complete before returning a result. Use with fewer than 10 records; for larger batches prefer the async `/run` endpoint. Like `/run`, it accepts at most 100 input records. If the run takes longer than 3 minutes, the endpoint returns `408` with the run id in the error detail; the run continues, and you can recover the result from the `/check` endpoint.
