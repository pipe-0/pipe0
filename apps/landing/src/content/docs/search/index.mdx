## Making requests

This request fetches a list of people using the [`people:profiles:crustdata@1`](/docs/search/search-catalog/people:profiles:crustdata/1) dataset.

```typescript tab="Typescript"
const result = await fetch("https://api.pipe0.com/v1/search/run", {
    method: "POST",
    headers: {
        "Authorization": `Bearer ${API_KEY}`,
        "Content-Type": "application/json"
    },
    body: JSON.stringify({
        search: {
            search_id: "people:profiles:crustdata@1",
            config: {
                limit: 100,
                filters: {
                    current_job_titles: ["Software Engineer", "Developer"]
                }
            }
        },
    })
});
```

You can find the list of available filters per dataset in the <AppLink linkType="searchCatalog">search catalog</AppLink>.

## Getting the task result

The request to `POST https://api.pipe0.com/v1/search/run` will create a search task.
To get the result of the task, poll the `GET https://api.pipe0.com/v1/search/check/{task_id}` endpoint until the task status is
`completed`.

```typescript tab="Typescript"
const result = await fetch("https://api.pipe0.com/v1/search/check/${taskId}", {
    method: "GET",
    headers: {
        "Authorization": `Bearer ${API_KEY}`
    }
});
```

```python tab="Python"
import requests
import json

headers = {
    "Authorization": f"Bearer {API_KEY}",
}
response = requests.get(f"https://api.pipe0.com/v1/search/check/{task_id}", headers=headers)
```

```bash tab="cURL"
curl -X GET "https://api.pipe0.com/v1/search/check/$TASK_ID" \
-H "Authorization: Bearer $API_KEY"
```

## Getting the result without polling

Instead of creating your task with the endpoint `POST https://api.pipe0.com/v1/search/run` use the endpoint `POST https://api.pipe0.com/v1/search/run/sync`.
Use the same payload for both endpoints. The `/sync` endpoint will try to return the `completed` result in one round-trip.

Read more about the trade-offs between <AppLink linkType="async">async vs sync requests</AppLink> here.
