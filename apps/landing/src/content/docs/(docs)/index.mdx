pipe0 is a data enrichment API. You send records (people, companies) and get them back with new fields: email addresses, phone numbers, LinkedIn profiles. Searches create records, pipes enrich them.

If all you need is an email or a phone number, this page is enough. Browse the <AppLink linkType="pipeCatalog">pipe catalog</AppLink> or <AppLink linkType="searchCatalog">search catalog</AppLink>, copy a request, and go.

<Callout>
You need an API key to run the examples. Create one in the dashboard, see <AppLink linkType="authentication">Authentication</AppLink>. All examples run in `sandbox` mode: mock data, zero credits, no subscription required.
</Callout>

## Find a work email

Send one record and one pipe. The `person:workemail:waterfall@1` pipe reads `name` and `company_domain` and adds a `work_email` field. You don't configure anything; the pipe knows its inputs and uses the default provider order.

export const enrichPayload = {
  config: { environment: "sandbox" },
  pipes: [{ pipe_id: "person:workemail:waterfall@1" }],
  input: [{ id: "1", name: "Jane Doe", company_domain: "pipe0.com" }],
};

<RequestExample kind="pipes" payload={enrichPayload} />

The response contains your record with the new field:

```json title="Response (truncated)"
{
  "id": "xgvxuqk2771d8t17n60vphu9",
  "status": "completed",
  "records": {
    "1": {
      "id": "1",
      "fields": {
        "work_email": {
          "type": "string",
          "value": "jane@pipe0.com",
          "status": "completed"
          // ...
        }
        // ...
      }
    }
  }
  // ...
}
```

Sandbox values are mock data. Remove `config.environment` (or set it to `"production"`) to run real providers and get real results.

## Search for people

While pipes add fields to records you already have, search creates the records. Use it when you start from a filter ("software engineers in Berlin") instead of a list.

export const searchPayload = {
  config: { environment: "sandbox" },
  search: {
    search_id: "people:profiles:crustdata@2",
    config: {
      limit: 25,
      filters: {
        current_job_titles: { include: ["Software Engineer"] },
      },
    },
  },
};

<RequestExample kind="search" payload={searchPayload} />

The `results` array of a search response is a valid `input` for a pipes request, so you can search for people and enrich them in two calls. See [Search then enrich](/docs/examples/search-then-enrich).

## Compose pipes

Stack pipes in one request. Each pipe reads the output of the pipes before it:

export const composePayload = {
  config: { environment: "sandbox" },
  pipes: [
    { pipe_id: "person:name:join@1" },
    { pipe_id: "person:workemail:waterfall@1" },
  ],
  input: [
    { id: "1", first_name: "Jane", last_name: "Doe", company_name: "Pipe0" },
  ],
};

<RequestExample kind="pipes" payload={composePayload} />

The second pipe read `name` even though your input never contained it: `person:name:join@1` created it from `first_name` and `last_name`. Pipes see each other's output. Two pipes can do what one can't.

## Motivation

When [Ivan Zhao](https://www.linkedin.com/in/ivanhzhao) explains the idea behind [Notion](https://www.notion.com/) he compares it to paper:

> Paper is simple enough for a 3-year-old to take out of the drawer and paint on it. Yet it's versatile enough that the bureaucracies of entire countries
> can run on it.

pipe0's philosophy is similar. One request finds an email address, and the same primitives are enough to build a Clay-like product. The <AppLink linkType="dough">Data tables example</AppLink> is an interactive enrichment table built in about 400 lines of code.
