This example takes a list of LinkedIn profile URLs and, in a single request:

1. Enriches the full LinkedIn profile for each person (`person:profile:waterfall@1`).
2. Uses an AI prompt to decide whether they are an ICP fit (`prompt:run@1`).
3. For matches only, resolves a work email (`person:workemail:profileurl:waterfall@1`), drafts a
   personalized email (`email:write@1`), and sends it (`email:send:gmail@1`).

The last three pipes are gated with [`run_if`](/docs/pipes/run-if), so non-matches never trigger an enrichment, a draft, or a send.

export const qualifyAndEmailPayload = {
  config: { environment: "production" },
  pipes: [
    // 1. Enrich the full LinkedIn profile from the profile URL.
    { pipe_id: "person:profile:waterfall@1" },
    // 2. Decide if the person is an ICP fit, based on the enriched profile.
    {
      pipe_id: "prompt:run@1",
      config: {
        model: "google-low",
        prompt: {
          template: `
You are qualifying a person against our Ideal Customer Profile (ICP).

[Our ICP]
Go-to-market, RevOps, or founder/leadership roles at B2B software companies that would
benefit from data enrichment (sales intelligence, prospecting, lead generation).

[The person]
{{ profile }}

[Output]
{% output is_icp_fit, type: "boolean", description: "true only if this person matches the ICP" %}
{% output reason, type: "string", description: "one short sentence justifying the decision" %}
          `,
        },
      },
    },
    // 3. ICP matches only: resolve a work email from the profile URL.
    {
      pipe_id: "person:workemail:profileurl:waterfall@1",
      run_if: {
        action: "run",
        when: {
          logic: "and",
          conditions: [
            { field_name: "is_icp_fit", property: "value", operator: "eq", value: true },
          ],
        },
      },
    },
    // 4. ICP matches only: draft a personalized email.
    {
      pipe_id: "email:write@1",
      run_if: {
        action: "run",
        when: {
          logic: "and",
          conditions: [
            { field_name: "is_icp_fit", property: "value", operator: "eq", value: true },
          ],
        },
      },
      config: {
        model: "google-low",
        persona: "sales",
        signature: "Jane from pipe0",
        template: `
Write a short, friendly outbound email introducing pipe0, an enrichment framework that helps
teams add data enrichment into their own products and go-to-market workflows.

Personalize it using what we know about the person:
{{ profile }}

Reason they are a good fit: {{ reason | default: "" }}

Keep it under 120 words and end with a soft call to action to book a quick call.
        `,
      },
    },
    // 5. Send the drafted email, but only once we actually have a work email.
    {
      pipe_id: "email:send:gmail@1",
      run_if: {
        action: "run",
        when: {
          logic: "and",
          conditions: [
            { field_name: "is_icp_fit", property: "value", operator: "eq", value: true },
            { field_name: "work_email", property: "status", operator: "eq", value: "completed" },
          ],
        },
      },
      connector: {
        strategy: "first",
        connections: [{ type: "vault", connection: "gmail_abcd123" }],
      },
      // To, subject, and body are templated. Reference the upstream fields.
      config: { to: "{{ work_email }}", subject: "{{ email_subject }}", body: "{{ email_body }}" },
    },
  ],
  input: [{ id: 1, profile_url: "https://www.linkedin.com/in/<HANDLE>" }],
};

How the pipes connect:

- `person:profile:waterfall@1` outputs a `profile` object. The AI prompt and the email writer
  both reference it with `{{ profile }}`.
- `email:write@1` produces `email_subject` and `email_body`; the send pipe wires them into its
  templated `subject` and `body` config fields.
- `email:send:gmail@1` needs a Gmail connection from your vault. Replace
  `gmail_abcd123` with your connection ID, copied exactly as the vault lists it
  (see [Connections](/docs/connections)).
- Swap `email:send:gmail@1` for `email:send:resend@1` if you send through Resend instead. That pipe
  also takes a `from` address.
