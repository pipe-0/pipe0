The pipe0 MCP server lets you search and enrich data directly from any MCP-compatible client. This includes
Claude, ChatGPT, Claude Code, Cursor, and custom agents.

The server exposes pipe0's <AppLink linkType="pipeCatalog">enrichment</AppLink> and <AppLink linkType="searchCatalog">search</AppLink> capabilities as MCP tools.

```txt title="MCP server URL"
https://api.pipe0.com/v1/mcp
```

## Authentication

The MCP server authenticates with OAuth. When you connect, your client opens a browser window where you
sign in to pipe0 and approve access. You never enter a client ID or client secret. The server registers
clients automatically through dynamic client registration.

<Callout>
  API keys do not work for the MCP server. They keep working for the REST API. If your client
  configuration sends an `Authorization` header with an API key, remove the header and reconnect
  with OAuth.
</Callout>

## Claude and Cowork

Connectors added on claude.ai are available in the Claude web app, the desktop app, and Cowork.

<Steps>
<Step>
Open **Settings > Connectors** and click **Add custom connector**.
</Step>
<Step>
Enter a name, for example `pipe0`, and set the remote MCP server URL to `https://api.pipe0.com/v1/mcp`.
Leave the OAuth client ID and secret under **Advanced settings** empty.
</Step>
<Step>
Click **Add**, then **Connect**. Sign in to pipe0 in the browser window and approve access.
</Step>
</Steps>

## Claude Code

Add the server from your terminal:

```bash title="Add the pipe0 MCP server"
claude mcp add --transport http pipe0 https://api.pipe0.com/v1/mcp
```

Run `/mcp` inside Claude Code, select **pipe0**, and choose **Authenticate**. Claude Code opens the
browser for the OAuth flow and stores the token.

## Cursor

Add the server to `~/.cursor/mcp.json`, or to `.cursor/mcp.json` inside a project:

```json title=".cursor/mcp.json"
{
  "mcpServers": {
    "pipe0": {
      "url": "https://api.pipe0.com/v1/mcp"
    }
  }
}
```

Open **Settings > MCP**. Cursor shows a login prompt next to `pipe0`. Click it to sign in and approve
access.

## ChatGPT

Custom MCP connectors are available on ChatGPT Business, Enterprise, and Edu plans. On other plans they
require developer mode, which you enable under **Settings > Apps & Connectors > Advanced settings**.

<Steps>
<Step>
Open **Settings > Apps & Connectors** and click **Create**.
</Step>
<Step>
Enter a name, set the MCP server URL to `https://api.pipe0.com/v1/mcp`, and pick **OAuth** as the
authentication method.
</Step>
<Step>
Create the connector, then sign in to pipe0 in the browser window and approve access.
</Step>
</Steps>

## Other MCP clients

Any client that supports the streamable HTTP transport and OAuth can connect with this configuration:

```json title="MCP configuration"
{
  "mcpServers": {
    "pipe0": {
      "url": "https://api.pipe0.com/v1/mcp"
    }
  }
}
```

The client discovers the OAuth endpoints from the server and starts the sign-in flow on first use.

## Available tools

The MCP server provides tools for:

- **Searching**: find people and companies using filters. Same capabilities as the <AppLink linkType="searchCatalog">search catalog</AppLink>.
- **Enriching**: find emails, phone numbers, company data, and more. Same capabilities as the <AppLink linkType="pipeCatalog">pipe catalog</AppLink>.

All MCP requests are <AppLink linkType="billing">billed</AppLink> like regular API requests.
