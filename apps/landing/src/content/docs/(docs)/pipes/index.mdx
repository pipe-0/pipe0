Pipes add fields to your input records: a work email, a phone number, a full company profile. You pick pipes from the <AppLink linkType="pipeCatalog">pipe catalog</AppLink>, send them with your input, and read the new fields off the response.

<YoutubeEmbed href="https://youtube.com/embed/7H7kHJgwSOg" layout="full" />

## Run a pipe

This request enriches one <AppLink linkType="inputObject">input object</AppLink>. The pipe `person:workemail:waterfall@1` reads `name` and `company_domain` and adds a `work_email` field.

export const runPayload = {
  pipes: [{ pipe_id: "person:workemail:waterfall@1" }],
  input: [{ id: "1", name: "Jane Doe", company_domain: "pipe0.com" }],
};

<RequestExample kind="pipes" payload={runPayload} />

The response contains one record per input object:

```json title="Response (truncated)"
{
  "id": "xgvxuqk2771d8t17n60vphu9",
  "status": "completed",
  "records": {
    "1": {
      "id": "1",
      "fields": {
        "work_email": {
          "type": "string",
          "value": "jane@pipe0.com",
          "status": "completed"
          // ...
        }
        // ...
      }
    }
  }
  // ...
}
```

The full response carries more metadata than shown here. See the <AppLink linkType="pipelineResponse">response object</AppLink>.

## Stack pipes

Stack pipes in one request to build workflows. Each pipe reads the output of the pipes before it:

export const stackPayload = {
  pipes: [
    { pipe_id: "person:name:join@1" },
    { pipe_id: "person:workemail:waterfall@1" },
  ],
  input: [
    { id: "1", first_name: "Jane", last_name: "Doe", company_name: "Pipe0" },
  ],
};

<RequestExample kind="pipes" payload={stackPayload} />

`person:name:join@1` creates `name` from `first_name` and `last_name`; the email pipe then reads `name` even though your input never contained it. Add <AppLink linkType="pipeline">`run_if` conditions</AppLink> to run pipes only for records that match, or <AppLink linkType="templateTags">templates</AppLink> to feed record fields into LLM prompts.

## Enrich in bulk

The sync endpoint above waits for the result and suits small batches (fewer than 10 records). For larger inputs, create an async task with `POST /v1/pipes/run` (same payload), then poll `GET /v1/pipes/check/{run_id}` until `status` is `completed`.

```typescript tab="TypeScript" title="Poll for the result"
const result = await fetch(`https://api.pipe0.com/v1/pipes/check/${runId}`, {
    method: "GET",
    headers: {
        "Authorization": `Bearer ${API_KEY}`
    }
});
```

```python tab="Python" title="Poll for the result"
import requests

headers = {
    "Authorization": f"Bearer {API_KEY}",
}
response = requests.get(f"https://api.pipe0.com/v1/pipes/check/{run_id}", headers=headers)
```

```bash tab="cURL" title="Poll for the result"
curl -X GET "https://api.pipe0.com/v1/pipes/check/$RUN_ID" \
-H "Authorization: Bearer $API_KEY"
```

Read more about the trade-offs in <AppLink linkType="async">sync vs async</AppLink>. The [TypeScript client](/docs/sdks/typescript-client) handles batching and polling for you.
