# Searches Quickstart

`Searches` is our endpoint for finding companies and people.
With searches, you can combine and deduplicate datasets from multiple providers.

If you already have a list of people or companies that you want to enrich, skip to <AppLink linkType="pipesQuickstart">pipes</AppLink>.


## Search Catalog

With searches, you gain access to multiple datasets via a single API endpoint. The <AppLink linkType="searchCatalog">search catalog</AppLink> 
lists all available datasets.

## Making Requests

Here's a request to fetch a list of companys using the search [`people:profiles:icypeas@1`]() dataset.

<CodeTabs items={["Typescript"]}>
    <div>
```typescript
const result = await fetch("https://api.pipe0.com/v1/searches/run", {
    method: "POST",
    headers: {
        "Authorization": `Bearer ${API_KEY}`,
    },
    body: JSON.stringify({
        config: {
            environment: "production",
            dedup: {
                strategy: "default"
            }
        },
        searches: [
            {
                search_id: "people:profiles:icypeas@1",
                config: {
                    limit: 100,
                    filters: {
                        industry: {
                            include: ["Software Development"]
                        }
                    }
                }
            },
        ],
    })
});
```
    </div>
</CodeTabs>

Passing a top-level `config` object is optional. The default environemnt is `production` and the default deduplication strategy is `default`.

The example uses one search but you are free to append multiple searches to the array of `searches`.

The result will look something like this:

```json

{
	"results": [
		{
			"name": {
				"value": "Gayle Pouros",
                "type": "string",
				"format": "text",
				"status": "completed"
				"ui": {},
			},
			"job_title": {
				"value": "Product Intranet Manager",
                "type": "string",
				"format": "text",
				"status": "completed",
				"ui": {},
			},
            ...
		},
		...
	]
}
```

> [!NOTE]
>
> The response will contain more information than displayed here. Learn how to work with <AppLink linkType="pipelineResponse">response objects</AppLink>.

The result of a search can be the input for a <AppLink linkType="pipesQuickstart">pipes request</AppLink>. Under the hood this uses <AppLink linkType="inputExpansion">input expansion</AppLink>.
