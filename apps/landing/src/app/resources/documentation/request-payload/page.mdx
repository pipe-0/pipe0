# Request Payload

The following API operations require a request payload of type `PipelineRequest`:

- `POST https://api.pipe0.com/v1/run` (create an enrichment task)
- `POST https://api.pipe0.com/v1/run/sync` (create an enrichment task return the result)

Here's a sample payload:

```json
{
  "pipes": [
    { 
      "pipe_id": "company:identity@1",
      "config": {
        "input_fields": {
          "company_name": {
            "alias": ""
          }
        }
      }
    },
  ],
  "input": [
    {
      "id": 1,
      "name": "Tom Schmidt",
      "company_name": "Pipe0",
    },
  ],
}
```


## Request payload properties

### pipes[]

An array of enrichment pipes alongside the pipe configuration. A full list of available pipes can be found in the 
<AppLink linkType="pipeCatalog">pipe catalog</AppLink>.

#### pipe.pipe_id

The unique identifier of a pipe.

#### pipe.config

Pipe config object. The full config object can be found in the <AppLink linkType="pipeCatalog">pipe catalog</AppLink>.
Passing pipe configuration is optional. If no configuration is passed, the default config is used. When a pipe offers
a `pipe.config.providers` option, the order of providers will be the order of processing.

#### pipe.config.input_fields

Pipe configuration for the input fields of a pipe.

#### pipe.config.input_field.alias

Defines a pointer for the field value. 

Example: Your input data contains a property called `custom_company_name`. You want to use the property `custom_company_name`
for the input field `comapny_name` of the pipe `company:identity@1`. To make this work, point the pipe input field to your
custom field by providing `"alias": "custom_company_name"` in the pipe config.

#### pipe.config.output_fields

Pipe configuration for the output fields of a pipe.

#### pipe.config.output_field.alias

Rename the pipe output field to a different `field_name`. This can be useful if you have two pipes that output the same field (error).

To avoid a duplicate output field rename the output field of one pipe to a new `fieldName`.

### input[]

An array of input objects. This is the data you want to enrich.

#### input.id

Each input object must have a unique id. The id can be a `string` or a `number`. Id's can be as simple as `1`, `2`, `3`, ...

#### input[number]

Each object in the `input` array represents an input object.

#### input[fieldName]

Each property of an input object represents an input field. Input fields are used as pipe input after 
<AppLink linkType="inputSanitation">input sanitation</AppLink>.





## OpenAPI 3.1

When creating request payloads it is helpful to have autocomplete in your IDE. For this, you can generate a client from 
our open API file hosted at:

import { Tabs } from 'nextra/components'

<Tabs items={['url']}>
  <Tabs.Tab>https://api.pipe0.com/openapi</Tabs.Tab>
</Tabs>

API clients wrap the HTTP API with language-native function calls.

#### TypeScript

#### Java / Kotlin
