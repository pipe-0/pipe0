## Sandbox Environment

Running pipelines in production incurs usage costs. This is why all pipes offer a 
<AppLink linkType="testMode">sandbox mode</AppLink>. **Making requests within the sandbox environment is always free.**

Expect the sandbox data to match production data structurally. Don't expect sandbox data to make sense.

To send sandbox requests, include the `"x-environment": "sandbox"` header.

<CodeTabs items={["Typescript", "Python", "cURL"]}>
    <div>
```typescript
const result = await fetch("https://api.pipe0.com/v1/run", {
method: "POST",
headers: {
    "authorization": `Bearer ${API_KEY}`,
    "x-environment": "sandbox" // default value is 'production'
},
body: JSON.stringify({
    pipes: [
    {
        pipe_id: "people:workemail:waterfall@1"
    },
    ],
    input: [
        {
            id: 1,
            name: "John Doe",
            company_name: "Google LLC"
        }
    ]
})
});
```

import requests
